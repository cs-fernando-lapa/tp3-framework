Frameworks são coleções que facilitam o processo de desenvolvimento.

Ele funciona como um conjunto de funções, que englobam funções maiores e complexas.

Podemos utilizar a seguinte analogia, você tem uma chave de fenda e quer parafusar o seu parafuso em algum lugar. O framework seria utilizar uma parafusadora elétrica, sendo assim, ainda iríamos parafusar o parafuso, porém com menos complexidade, tempo e mais agilidade para dedicarmos tempo em outro processo.

Utilizaremos ele no projeto pra deixar o mesmo mais ágil e melhor e padronizado.

O framework é pra deixar o projeto mais barato, desde que seja mapeado todo o projeto e qual o melhor framework a ser utilizado no mesmo.

A estrutura da aplicação é padronizada no MVC(Model, View, Controller). Basicamente, essa estrutura divide sua aplicação pra facilitar em modificação paralelas.

O Model é basicamente a representação dos dados. Ex.: Cadastrar usuário -> nome : Cliente Nesse caso o model é o dado “Cliente”.

O View é literalmente a sua página. Onde os Model serão apresentados. O Controller(á respondendo a segunda pergunta), é o controle dos Model’s, e onde eles serão apresentados nas View’s.

Basicamente, podemos citar o seguinte exemplo no controller: Ex.: Na View temos a tela de Editar o cadastro do usuário Nela tem o campo nome, onde o Controller disponibiliza o “nome” ao campo na View, pegando a informação no Model.

As diretivas basicamente são implementações de funções, porém de forma declarativa. Filtro serve para a view limitar o que será exibido do controller. O serviço faz o pré processamento dos dados, e os prepara para serem consumidos pelo controller.
